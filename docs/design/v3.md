# Icon Pipeline v3: Source SVG/PNG → ICNS + PNG Sets

This document defines a **complete, reproducible icon build pipeline** for macOS apps, web apps, plugins, and extensions.

It includes:
- Repository layout
- `Makefile`
- Build scripts (with SVG + PNG support)
- AI prompt spec for generating new source icons

The goal is **one source icon → many correct outputs**, with minimal tooling and zero guesswork.

**What's new in v3:**
- SVG source support — each output size is rendered directly from SVG for maximum sharpness
- Renamed "master" → "source" throughout
- Scripts extracted into real files (not just spec)

---

## 1. Repository Layout

```
generate-icons/
├── Makefile
├── icons/                     # source icons (1024x1024 PNG or SVG)
│   ├── app.png
│   └── logo.svg
├── dist/                      # generated outputs (gitignored)
│   ├── icns/
│   └── png/
├── docs/
│   └── ai-source-icon-prompt.md
└── scripts/
    ├── build-all.sh
    ├── build-icns.sh
    ├── build-png-set.sh
    ├── svg-to-png.sh
    └── verify-sources.sh
```

---

## 2. Requirements

- macOS (for `iconutil` and `sips`)
- bash
- `rsvg-convert` (only if using SVG sources) — `brew install librsvg`

> PNG resizing uses `sips`. SVG rendering uses `rsvg-convert` for per-size rasterization. ICNS creation uses `iconutil`.

---

## 3. Source Icon Requirements

### PNG sources
- Exactly 1024×1024
- RGBA with transparency
- Strong silhouette at 16–32px
- Avoid thin strokes and tiny details
- ~10–15% padding (safe area)

### SVG sources
- Valid SVG file
- Square aspect ratio recommended
- Simple geometry for clean rasterization at small sizes

---

## 4. Commands

```sh
make all        # verify + build ICNS + PNG sets from all sources
make icns       # build only ICNS files
make png        # build only PNG sets
make verify     # validate all source icons
make clean      # wipe dist/ outputs
make check-deps # check tool availability
```

---

## 5. Output Sizes

**ICNS** (macOS app icons):
- 16, 32, 64, 128, 256, 512, 1024 (+ @2x retina variants)

**PNG sets** (web/extensions):
- 16, 32, 48, 128, 256

---

## 6. SVG Rendering Strategy

When the source is SVG, each output size is rendered **directly from the SVG** using `rsvg-convert`. This produces sharper results at small sizes compared to downscaling from a single 1024px rasterization.

When the source is PNG, the existing `sips` downscale pipeline is used (unchanged from v2).
