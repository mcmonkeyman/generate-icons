## 2. README.md

# Icon Pipeline (master PNG → ICNS + PNG sets)

A small, deterministic build pipeline for icons.

## What this repo does

Input:
- `icons/*.png` (each **1024×1024 PNG** with **transparent background**)

Outputs:
- `dist/icns/*.icns` (macOS app icons)
- `dist/png/<name>/*.png` (common PNG sizes for web/extensions)

## Requirements

- macOS (for `iconutil` and `sips`)
- bash (default on macOS)

> PNG resizing works with `sips`. ICNS creation uses `iconutil` (Apple’s tool), which is the most reliable way to preserve alpha/transparency.

## Quick start

1) Add one or more master icons to `icons/`:
- `icons/app.png`
- `icons/tray.png`

2) Build everything:
```sh
make all

4. Scripts

scripts/verify-masters.sh

#!/usr/bin/env bash
set -euo pipefail

ICON_DIR="${1:-icons}"

if [[ ! -d "$ICON_DIR" ]]; then
  echo "ERROR: icon dir not found: $ICON_DIR" >&2
  exit 1
fi

shopt -s nullglob
files=("$ICON_DIR"/*.png)
if (( ${#files[@]} == 0 )); then
  echo "ERROR: no PNG masters found in $ICON_DIR" >&2
  exit 1
fi

fail=0

for f in "${files[@]}"; do
  w=$(sips -g pixelWidth "$f" 2>/dev/null | awk -F': ' '/pixelWidth/ {print $2}' | tr -d ' ')
  h=$(sips -g pixelHeight "$f" 2>/dev/null | awk -F': ' '/pixelHeight/ {print $2}' | tr -d ' ')

  if [[ "$w" != "1024" || "$h" != "1024" ]]; then
    echo "✗ $f  (expected 1024x1024, got ${w:-?}x${h:-?})"
    fail=1
  else
    echo "✓ $f  (1024x1024)"
  fi
done

if (( fail )); then
  echo
  echo "One or more masters are not 1024x1024. Fix them before building."
  exit 2
fi

scripts/build-icns.sh

#!/usr/bin/env bash
set -euo pipefail

IN_PNG="${1:?usage: build-icns.sh <input.png> <output.icns>}"
OUT_ICNS="${2:?usage: build-icns.sh <input.png> <output.icns>}"

mkdir -p "$(dirname "$OUT_ICNS")"

SIZES=(16 32 64 128 256 512 1024)

ICONSET_DIR="$(mktemp -d)"
trap 'rm -rf "$ICONSET_DIR"' EXIT

for SIZE in "${SIZES[@]}"; do
  sips -z "$SIZE" "$SIZE" "$IN_PNG" \
    --out "$ICONSET_DIR/icon_${SIZE}x${SIZE}.png" >/dev/null

  if (( SIZE < 1024 )); then
    RETINA=$((SIZE * 2))
    if (( RETINA <= 1024 )); then
      sips -z "$RETINA" "$RETINA" "$IN_PNG" \
        --out "$ICONSET_DIR/icon_${SIZE}x${SIZE}@2x.png" >/dev/null
    fi
  fi
done

iconutil -c icns "$ICONSET_DIR" -o "$OUT_ICNS"
echo "Wrote $OUT_ICNS"

#!/usr/bin/env bash
set -euo pipefail

IN_PNG="${1:?usage: build-png-set.sh <input.png> <output_dir>}"
OUT_DIR="${2:?usage: build-png-set.sh <input.png> <output_dir>}"

mkdir -p "$OUT_DIR"

SIZES=(16 32 48 128 256)

for SIZE in "${SIZES[@]}"; do
  sips -z "$SIZE" "$SIZE" "$IN_PNG" \
    --out "$OUT_DIR/${SIZE}.png" >/dev/null
done

echo "Wrote PNG set: $OUT_DIR"

scripts/build-all.sh

#!/usr/bin/env bash
set -euo pipefail

MODE="${1:-all}" # all | icns | png

SRC_DIR="icons"
OUT_DIR="dist"

mkdir -p "$OUT_DIR/icns" "$OUT_DIR/png"

shopt -s nullglob
ICONS=("$SRC_DIR"/*.png)

if (( ${#ICONS[@]} == 0 )); then
  echo "ERROR: no masters found in $SRC_DIR" >&2
  exit 1
fi

for ICON in "${ICONS[@]}"; do
  NAME="$(basename "$ICON" .png)"

  case "$MODE" in
    all)
      ./scripts/build-icns.sh "$ICON" "$OUT_DIR/icns/$NAME.icns"
      ./scripts/build-png-set.sh "$ICON" "$OUT_DIR/png/$NAME"
      ;;
    icns)
      ./scripts/build-icns.sh "$ICON" "$OUT_DIR/icns/$NAME.icns"
      ;;
    png)
      ./scripts/build-png-set.sh "$ICON" "$OUT_DIR/png/$NAME"
      ;;
    *)
      echo "ERROR: unknown mode: $MODE" >&2
      exit 2
      ;;
  esac
done

# AI Master Icon Prompt Spec (1024×1024 transparent PNG)

## Goal
Generate a single master icon that:
- Scales cleanly to 16px
- Works for macOS ICNS and web PNG sets
- Has a strong, simple silhouette

## Constraints
- PNG, 1024×1024
- Transparent background
- No text, no letters, no watermark
- Minimal detail
- Centered composition
- ~10–15% padding (safe area)

## Prompt Template

Create a 1024×1024 transparent PNG app icon.

Subject: <describe the symbol or metaphor>
Brand tone: <developer tool / professional / friendly>
Style: flat or lightly shaded, geometric
Detail level: low
Avoid thin strokes and micro-details
No background plate unless specified
No text or lettering
High contrast
Centered composition
Transparent background

Output:
- Transparent PNG
- 1024×1024
- Crisp edges

## Example

Create a 1024×1024 transparent PNG app icon.

Subject: a stylized portal made of two overlapping rounded rectangles forming a doorway.
Brand tone: modern developer tool, calm and confident.
Style: geometric, minimal.
No text or watermark.
Leave ~12% padding.
High contrast.
Transparent background.

Deliver as a transparent 1024×1024 PNG.

## Validation Checklist
- Does it read at 16px?
- Does it remain crisp at 32px?
- Are shapes bold enough?
- Is contrast sufficient?
